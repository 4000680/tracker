# Дизайн-бриф: «Картотека рассылки» — Telegram Mini App

## 1. Что это за продукт

Личный рабочий инструмент Владимира — предпринимателя в сфере таможенного оформления автомобилей (утильсбор, СБКТС, ЭПТС). Приложение — трекер холодной рассылки по контактам с Авито: каждый день нужно связаться с 5 контактами из каждого списка, отправить оффер, зафиксировать статус и ответ. Открывается внутри Телеграма как Mini App, используется в основном с телефона, десятки раз в день, короткими сессиями по 1–3 минуты.

Единственный пользователь — сам владелец. Не витрина и не продукт на продажу: это рабочий станок. Дизайн должен помогать быстро работать, а не впечатлять.

## 2. Структура приложения (3 экрана, 3 HTML-файла)

### Экран 1 — index.html: выбор списка
Стартовый экран. Заголовок, текущая дата, две большие карточки-кнопки:
- **Брокеры** — коллеги-брокеры, 240 контактов (партнёрская рассылка: «скидывайте заказы, отработаю, поделимся»)
- **Автосалоны** — заказчики-перегонщики и автосалоны, 720 контактов (клиентская рассылка: продажа услуги оформления)

Внизу — короткая подсказка о хранении данных в облаке Телеграма и кнопке переноса данных.

### Экраны 2 и 3 — brokery.html и avtosalony.html: трекеры списков
Оба трекера устроены одинаково, отличаются данными и текстами офферов. Элементы интерфейса:

- **Шапка**: название трекера, подпись с числом контактов, бейдж с общим количеством.
- **Навигация по дням**: стрелки влево/вправо, название и дата дня, кнопка «Сегодня». Все контакты разбиты по дням, по 5 контактов на день (в списке автосалонов — 144 дня).
- **Прогресс дня**: полоса прогресса и счётчик «отправлено X из 5».
- **Фильтр по статусам**: выпадающий список / вкладки: Все, Ответил, Переговоры, Договорились и т.д.
- **Блок «Все тексты»** (сворачиваемый): готовые тексты офферов, «крючков» и сообщений для Телеграма с кнопками «Скопировать».
- **Скрипт звонка** (в трекере автосалонов): пошаговый сценарий разговора — представление, зацепка, суть, ценность, закрытие, отработка возражений. Каждый шаг с кнопкой копирования.
- **Карточки контактов** (главный элемент, 5 на день): имя продавца, контактное лицо, регион и город, число просмотров объявления, ссылка на объявление Авито, назначенный оффер (тег), выбор статуса (выпадающий список), под-статус для ответивших, поле заметки, отметка «звонил», кнопки «Скопировать текст».
- **Тост** — всплывающее уведомление «Текст скопирован».
- **Кнопка ⇅ (внизу слева)** — панель переноса данных: текстовое поле, кнопки «Выгрузить данные» и «Загрузить данные», ссылка «← К выбору списка».

## 3. Текущий визуальный стиль (можно переосмыслить)

Сейчас: светлый фон #F2F4F6, белые карточки, тёмно-синий акцент #1C4E80, красный «штамп» #B23A2E, зелёный для успеха #2F8F5B, шрифты IBM Plex Sans + IBM Plex Mono, скругления 14–16px, макет в одну колонку max-width 520px. Настроение — «картотека / документооборот»: табы-папки, штамп «КАРТОТЕКА», моноширинные даты и счётчики.

## 4. Задача дизайна

Придумать классный, цельный фирменный стиль для всех трёх экранов. Свобода полная — палитра, типографика, характер. Тематика бизнеса, от которой можно отталкиваться: таможня, документы, печати, автомобили, импорт, СБКТС/ЭПТС. Настроение: собранный рабочий инструмент профессионала, а не игрушка.

## 5. Жёсткие требования (не нарушать)

1. **Мобайл-прежде-всего**: ширина экрана ~380px внутри Телеграма, одна колонка, крупные зоны нажатия (минимум 40px), нижняя часть экрана не перекрывается системными кнопками Телеграма.
2. **Скорость работы важнее красоты**: статус ставится в 1 касание, текст копируется в 1 касание, переход между днями — в 1 касание. Никаких лишних шагов и модальных окон там, где их нет сейчас.
3. **Сохранить весь функционал и логику без изменений**: все перечисленные в разделе 2 элементы должны остаться. Если дизайн делается поверх готового кода — менять только CSS/оформление, не трогать JavaScript, id и обработчики, массив DATA с контактами, ключи хранилища (broker-status, perig-status) и скрипт-адаптер window.storage (Telegram CloudStorage).
4. **Каждый экран — один автономный HTML-файл** (стили внутри), без сборщиков, без внешних зависимостей кроме Google Fonts и telegram-web-app.js. Файлы размещаются на GitHub Pages.
5. **Не использовать localStorage/sessionStorage напрямую** — только существующий адаптер window.storage.
6. **Тексты интерфейса — на русском.** Названия файлов не менять: index.html, brokery.html, avtosalony.html.
7. Читаемость при ярком свете (использование на улице/в машине): достаточный контраст текста.

## 6. Технический контекст

- Платформа: Telegram Mini App (открывается через кнопку меню бота), также работает в обычном браузере.
- Хранение: облако Телеграма (CloudStorage) через встроенный адаптер; данные — один JSON-объект статусов на список.
- Кнопка «Назад» Телеграма ведёт с трекера на index.html.
- Хостинг: GitHub Pages, статические файлы.
